# Markdown TOC

마크다운 파일에 `<!-- TOC -->` / `<!-- /TOC -->` 마커 사이의 목차를 자동으로 생성하거나 갱신합니다.

## 언제 사용하나요

다음과 같은 요청이 들어올 때 사용합니다.

- "이 README에 TOC 만들어줘"
- "목차 갱신해줘"
- "table of contents 추가해줘"

## 동작 절차

1. **마커 찾기** — 대상 파일에서 `<!-- TOC -->` 와 `<!-- /TOC -->` 마커 쌍을 찾습니다.
2. **마커가 없으면 물어보기** — 마커 쌍이 없다면 사용자에게 "첫 번째 H1 바로 아래에 마커를 추가해도 될까요?" 라고 물어봅니다. 동의하면 추가하고, 아니면 작업을 중단합니다.
3. **헤더 수집** — 본문의 `##`, `###`, `####` 헤더를 위에서 아래로 순서대로 수집합니다.
   - `#` (H1)은 문서 제목이므로 목차에서 제외합니다.
   - ` ``` ` 으로 감싸인 코드 블록 안의 `#` 은 헤더가 아니므로 무시합니다.
4. **앵커 슬러그 생성** — 각 헤더 텍스트를 GitHub 스타일 슬러그로 변환합니다.
   - 모두 소문자화
   - 공백은 `-` 로 치환
   - 영숫자, 한글, 하이픈을 제외한 문자는 제거
   - 동일한 슬러그가 이미 있으면 `-1`, `-2`, ... 접미사를 붙여 충돌을 피합니다.
5. **마커 사이 교체** — `<!-- TOC -->` 와 `<!-- /TOC -->` 사이의 기존 내용을 새로 만든 목차로 통째로 교체합니다.

## 출력 형식

`##` 은 들여쓰기 0칸, `###` 은 2칸, `####` 은 4칸으로 표현합니다.

```markdown
<!-- TOC -->
- [소개](#소개)
  - [설치](#설치)
  - [요구사항](#요구사항)
- [사용법](#사용법)
  - [기본 사용](#기본-사용)
    - [옵션](#옵션)
- [라이선스](#라이선스)
<!-- /TOC -->
```

## 주의사항

- 코드 블록(``` ``` ```) 안의 `#` 은 헤더로 인식하지 않습니다.
- 같은 이름의 헤더가 여러 개라면 두 번째부터 `-1`, `-2` 가 붙습니다. 예: `## 예시` 가 두 번 나오면 두 번째 슬러그는 `예시-1` 이 됩니다.
- 마커 외부의 본문은 절대 수정하지 않습니다.
